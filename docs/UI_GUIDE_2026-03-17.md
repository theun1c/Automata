# UI Guide (Сессия 17.03.2026)

Этот файл для быстрого разбора того, что сделано по UI и где это менять.
Реализовано в режиме **UI-only**: без БД/API/бизнес-логики.

## 1) Как устроен Desktop UI

### Базовая связка
- `src/Automata.Desktop/App.axaml`
Комментарий: подключает `ViewLocator`, чтобы `ViewModel` автоматически сопоставлялся с `View` по имени.

- `src/Automata.Desktop/ViewLocator.cs`
Комментарий: правило соответствия `XxxViewModel -> XxxView`.

- `src/Automata.Desktop/ViewModels/MainWindowViewModel.cs`
Комментарий: главный роутер desktop-разделов.
Что важно:
- enum `DesktopSection` определяет разделы (`Главная`, `Монитор ТА`, `Администрирование` и т.д.).
- `_sections` хранит экземпляры страниц.
- `CurrentSection` переключает текущую страницу.
- `IsAdminExpanded` управляет раскрытием блока `Администрирование`.
- команды `Show*` и `ToggleAdminCommand` связаны с кнопками меню.

- `src/Automata.Desktop/Views/MainWindow.axaml`
Комментарий: основной shell (верхняя белая полоска, темный sidebar, черная верхняя панель контента, рабочая область).
Что важно:
- меню 1-го уровня: `Главная`, `Монитор ТА`, `Детальные отчеты`, `Учет ТМЦ`, `Администрирование`.
- `Администрирование` раскрывается/сворачивается.
- burger-иконка теперь в шапке sidebar (как на скриншоте).
- рендер страницы: `ContentControl Content="{Binding CurrentPage}"`.

- `src/Automata.Desktop/Views/MainWindow.axaml.cs`
Комментарий: стандартный code-behind окна.

## 2) Страницы Desktop

### Авторизация
- `src/Automata.Desktop/ViewModels/LoginViewModel.cs`
Комментарий: mock-состояния входа, переключатель показа пароля, ошибка.

- `src/Automata.Desktop/Views/LoginView.axaml`
Комментарий: визуал входа (email, пароль, показать/скрыть, кнопки).

- `src/Automata.Desktop/Views/LoginView.axaml.cs`
Комментарий: стандартный code-behind.

- `src/Automata.Desktop/ViewModels/RegistrationViewModel.cs`
Комментарий: mock-состояния регистрации, CAPTCHA, эмуляция кода подтверждения.

- `src/Automata.Desktop/Views/RegistrationView.axaml`
Комментарий: форма регистрации + модалка эмуляции кода.

- `src/Automata.Desktop/Views/RegistrationView.axaml.cs`
Комментарий: стандартный code-behind.

### Главная
- `src/Automata.Desktop/ViewModels/DashboardViewModel.cs`
Комментарий: mock-данные карточек, сети, новостей, динамики продаж.

- `src/Automata.Desktop/Views/DashboardView.axaml`
Комментарий: визуал кабинета (эффективность, состояние, сводка, динамика, новости).

- `src/Automata.Desktop/Views/DashboardView.axaml.cs`
Комментарий: стандартный code-behind.

### Монитор ТА (desktop-вариант)
- `src/Automata.Desktop/ViewModels/MonitorTaViewModel.cs`
Комментарий: mock-строки таблицы монитора в desktop.

- `src/Automata.Desktop/Views/MonitorTaView.axaml`
Комментарий: UI монитора (фильтры, сортировка, таблица, сводка).

- `src/Automata.Desktop/Views/MonitorTaView.axaml.cs`
Комментарий: стандартный code-behind.

### Администрирование: Торговые автоматы
- `src/Automata.Desktop/ViewModels/MachinesViewModel.cs`
Комментарий: mock-список ТА, фильтр, пагинация, таблица/плитка, команды формы и подтверждений.
Отвязка модема:
- `ConfirmUnbindModem()` ставит `-1` в UI.

- `src/Automata.Desktop/Views/MachinesView.axaml`
Комментарий: таблица/плитка ТА, фильтры и панель действий.

- `src/Automata.Desktop/Views/MachinesView.axaml.cs`
Комментарий: здесь реализовано открытие **отдельных модальных окон** (не оверлей):
- форма создания/редактирования,
- подтверждение удаления,
- подтверждение отвязки модема.
Механика:
- подписка на `PropertyChanged` VM,
- при `IsMachineFormOpen / IsDeleteConfirmOpen / IsUnbindConfirmOpen` открывается `ShowDialog(owner)`.

### Администрирование: Компании
- `src/Automata.Desktop/ViewModels/CompaniesViewModel.cs`
Комментарий: mock-список компаний, фильтр, пагинация, команды формы и удаления.

- `src/Automata.Desktop/Views/CompaniesView.axaml`
Комментарий: таблица/плитка компаний + фильтры.

- `src/Automata.Desktop/Views/CompaniesView.axaml.cs`
Комментарий: открытие отдельных модальных окон формы/удаления через `ShowDialog(owner)`.

## 5) Быстрая карта доработок до 100%

- Меню/иконки/отступы в desktop:
`src/Automata.Desktop/Views/MainWindow.axaml`

- Логика переключения разделов:
`src/Automata.Desktop/ViewModels/MainWindowViewModel.cs`

- Визуал конкретных страниц:
`src/Automata.Desktop/Views/*View.axaml`

- Модальные окна и их поведение:
`src/Automata.Desktop/Views/*Window.axaml` + `MachinesView.axaml.cs` / `CompaniesView.axaml.cs`

- Web shell и монитор:
`src/Automata.Web/Pages/Shared/_Layout.cshtml`,
`src/Automata.Web/wwwroot/css/site.css`,
`src/Automata.Web/Pages/Monitor/*`

## 6) Важное ограничение на сегодня

В этой среде `dotnet build` падает без диагностических ошибок (`0 errors`), поэтому финальную сборку здесь подтвердить нельзя.  
Но XAML-файлы проходят XML-валидацию, а UI-связки (bindings/команды/модалки) сведены и документированы выше.

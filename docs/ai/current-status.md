# Текущий подход разработки

Разработка начинается с проектирования архитектуры системы 

Причина:

Важно продумать как компоненты системы будут взаимодействовать друг с другом

Правильная архитектура обеспечивает скорость и удобство разработки, масштабируемость всей системы и ее поддержки

Необходимо сначала понять:

- какие независимые и зависимые компоненты есть в системе 
- как происходит взаимодействие компонентов системы
- какие сущности есть
- на какие отдельные фичи нужно разбить проект (по ТЗ)

Режим работы:
- Только UI-only.
- Никакой бизнес-логики, никакой БД, никаких API-вызовов.
- UI на русском.
- Не менять стек.

Задачи этой сессии:
1) Дожать дизайн desktop/web до максимального совпадения с assets.
2) Параллельно сделать архитектурный аудит текущей структуры.
3) Подготовить и начать аккуратный рефакторинг структуры (без массовых поломок).

Правила выполнения:
- Перед изменениями: покажи краткий план шага.
- После каждого шага: покажи, какие файлы изменены и что именно сделано.
- Делай атомарные коммиты по git-flow стилю (`feat/fix/refactor/docs/chore`).
- Не делать один большой коммит.
- Не трогать unrelated файлы.

Обязательные промежуточные отчеты:
- Текущий прогресс в 3-5 пунктов.
- Что осталось до завершения.
- Риски/блокеры.

Финальный отчет:
- список измененных файлов;
- что улучшено визуально;
- что сделано по архитектуре;
- что осталось на следующий шаг;
- список коммитов.

Если есть неоднозначность по макету — сначала спроси меня, потом вноси правки.

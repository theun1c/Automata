Работай строго в рамках UI-only этапа. Никакой бизнес-логики, никакой БД, никаких API-вызовов.

Цель: за одну сессию собрать полный визуальный каркас desktop + web по спецификации.

ОБЩИЕ ПРАВИЛА
- Не менять стек и существующую структуру проекта.
- UI только на русском языке.
- Использовать mock/stub данные внутри ViewModel/PageModel.
- Никаких EF-запросов, репозиториев, сервисов БД.
- Никакой реальной авторизации/регистрации, только визуальные состояния.
- Деструктивные действия (Удалить, Отвязать модем) показывать через модальные подтверждения (UI).
- После “отвязки модема” визуально показывать -1 (на mock-строке/демо-сценарии).
- По завершению: собрать проекты, показать список измененных файлов и покрытие требований UI.

ЧАСТЬ 1. DESKTOP (Avalonia) — UI-ONLY
=====================================

1) Навигация/каркас
- Обнови главное окно и структуру ViewModels так, чтобы были разделы:
    - Вход
    - Регистрация
    - Главная
    - Администрирование
        - Торговые автоматы
        - Компании
- Сохранить текущий стиль приложения, но довести макет до рабочего UI-каркаса.

2) Экран входа (UI)
   Состав:
- Поле Email
- Поле Пароль
- Переключатель показать/скрыть пароль
- Кнопка Вход
- Переход на регистрацию
- Место для текста ошибки (визуально)

3) Экран регистрации (UI)
   Состав:
- Email
- Пароль
- Поле кода подтверждения
- Поле кода франчайзи
- CAPTCHA (минимум 3 арифметические операции) — как текстовый challenge + поле ответа
- Кнопка «Зарегистрироваться»
- Модальное окно “Эмуляция кода подтверждения” (показывает mock код)
- Визуальные состояния:
    - пустая форма
    - ошибка email
    - ошибка пароля
    - неверная CAPTCHA
    - код сгенерирован
    - неверный код подтверждения
    - неверный код франчайзи
    - успешная регистрация
      (состояния можно переключать mock-флагами во ViewModel)

4) Главная страница (UI)
   Собери 5 блоков:
- Эффективность сети (процент)
- Состояние сети (визуальный блок-диаграмма/статусы)
- Сводка (деньги/выручка/инкассация/обслуживание)
- Динамика продаж за 10 дней (mock-график + переключатель “Сумма/Количество”)
- Новости (список)
  Все данные mock.

5) Администрирование → Торговые автоматы (UI)
   Список:
- Табличный вид (по умолчанию) + переключение на плитку
- Колонки:
  ID, Название автомата, Модель, Компания, Модем, Адрес/Место, В работе с, Действия
- Действия в строке:
  Редактировать, Удалить, Отвязать модем
- Элементы:
  фильтр по названию, пагинация, выбор количества строк, счетчик “N из M”, кнопка экспорта CSV
- Нечетные строки визуально выделить
- Для одной mock-строки modem = -1

Форма создания/редактирования ТА (UI):
- Все поля из CODEX_SPEC 10.2.1 должны присутствовать визуально
- Кнопки: Сохранить/Отмена
- Без реальной валидации и сохранения

6) Администрирование → Компании (UI)
   Список:
- Колонки:
  Название, Вышестоящая, Адрес, Контакты, В работе с, Действия
- Добавление/Редактирование/Удаление (через UI-элементы/модалки)
- Фильтр, выбор количества строк, счетчик “N из M”, экспорт CSV
- Таблица + плитка (если быстро, иначе таблица приоритет)

Форма компании (UI):
- Вышестоящая компания
- Название компании
- Адрес компании
- Контакты компании
- Примечания
- Кнопки: Сохранить/Отмена

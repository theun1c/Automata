# Информационная система для сети вендинговых аппаратов

Проект предназначен для управления сетью торговых автоматов, учета продаж и обслуживания, а также мониторинга состояния оборудования.

## Что включает проект

Система состоит из трех основных частей:

- **Desktop-приложение** для франчайзера;
- **Web-модуль «Монитор ТА»** для наблюдения за автоматами;
- **Реляционная база данных** со стартовыми тестовыми данными.

## Основные возможности

- регистрация франчайзера по `email + пароль`;
- подтверждение email кодом с эмуляцией отправки в модальном окне;
- проверка уникального кода франчайзи;
- авторизация и вход в личный кабинет;
- главная страница с виджетами состояния сети;
- администрирование торговых автоматов и компаний;
- мониторинг автоматов через web-интерфейс;
- фильтры, сортировки, пагинация и экспорт CSV;
- загрузка тестовых данных для демонстрации работы системы.

## Ключевые требования

- БД должна быть приведена минимум к **3НФ**;
- у пользователя должен быть уникальный email;
- пароль должен храниться в виде хеша;
- CAPTCHA при регистрации должна содержать минимум **3 арифметические операции**;
- после отвязки модема в интерфейсе и CSV должно отображаться значение **`-1`**;
- в проекте должны быть:
    - не менее **10 ручных тест-кейсов**;
    - не менее **15 unit-тестов** для регистрации и авторизации.

## Модули системы

### Desktop-приложение
- вход и регистрация;
- главная страница;
- раздел **«Администрирование»**;
- подразделы:
    - **«Торговые автоматы»**;
    - **«Компании»**.

### Web-модуль
- экран **«Монитор ТА»**;
- просмотр состояния автоматов;
- фильтрация, сортировка и экспорт данных.

## Базовый сценарий запуска

Точные команды запуска зависят от выбранного технологического стека. Базовая последовательность следующая:

1. Создать и настроить базу данных.
2. Применить миграции или SQL-скрипты.
3. Загрузить стартовые данные (`seed` / `import`).
4. Запустить backend/API.
5. Запустить desktop-приложение.
6. Запустить web-модуль.

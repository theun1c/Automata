# AGENTS.md

Этот репозиторий реализует информационную систему для сети вендинговых аппаратов.

## Главные источники требований
Перед любыми изменениями обязательно изучи:
1. `docs/CODEX_SPEC.md` — основной рабочий документ проекта.
2. `ЗАдание УП01 2026.docx` — исходное ТЗ.
3. Макеты и визуальные референсы в `assets/`.
4. ER-набросок в `docs/er/ER-excalidraw.excalidraw`.

## Базовый подход
- Не ломай существующий стек и структуру проекта без крайней необходимости.
- Не усложняй архитектуру и БД сверх требований ТЗ.
- Все новые сущности, поля и сценарии должны быть оправданы конкретным экраном, правилом или данными из ТЗ.
- Если текст ТЗ и макет расходятся, поля и видимые элементы бери из макета, а бизнес-правила — из ТЗ.

## Приоритеты реализации
1. Сначала рабочая БД, схема `automata`, seed/import и ограничения.
2. Затем auth/registration.
3. Затем desktop-модуль: Главная → Администрирование.
4. Затем web-модуль «Монитор ТА».
5. Затем тесты и документация.

## Обязательные правила
- UI должен быть на русском языке.
- Пароли хранить только в хеше.
- Реальную отправку email не делать: код подтверждения эмулировать в модальном окне.
- Для регистрации нужна математическая CAPTCHA минимум с 3 арифметическими операциями.
- Проверка email, пароля и кода франчайзи обязательна.
- После отвязки модема в UI и CSV показывать `-1`, даже если в БД хранится `NULL`.
- Если реальный API web-монитора недоступен, использовать mock/stub adapter с фиксированным контрактом.
- Для списков реализовать фильтр, пагинацию, экспорт CSV и корректное отображение количества записей.
- Деструктивные операции подтверждать модальным окном.
- Не добавлять лишние таблицы и справочники: БД должна соответствовать ТЗ и быть приведена к 3НФ без переусложнения.

## Минимальный ожидаемый результат
- БД в 3НФ на разумном уровне;
- desktop: вход, регистрация, Главная, Торговые автоматы, Компании;
- web: Монитор ТА;
- 10 ручных тест-кейсов;
- 15+ unit-тестов для авторизации и регистрации;
- README с инструкцией запуска.

## Что нельзя пропускать
- валидацию email и пароля;
- CAPTCHA;
- проверку кода франчайзи;
- экспорт CSV;
- сортировки и фильтры web-монитора;
- поведение отвязки модема;
- тесты.

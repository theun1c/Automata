# AGENTS.md

Этот репозиторий реализует информационную систему для сети вендинговых аппаратов.

## Главный документ
Основная спецификация находится в `docs/CODEX_SPEC.md`.  
Перед любыми изменениями сначала прочитай ее целиком.

## Приоритеты
1. Не ломай существующий стек и структуру проекта.
2. Сначала обеспечь работающую БД и seed/import.
3. Затем реализуй auth/registration.
4. Затем desktop-модуль: Главная → Администрирование.
5. Затем web-модуль «Монитор ТА».
6. Затем тесты и документацию.

## Обязательные правила
- Следуй требованиям из `docs/CODEX_SPEC.md`.
- Если текст ТЗ и макет расходятся, поля и видимые элементы бери из макета.
- UI должен быть на русском языке.
- Пароли хранить только в хеше.
- Реальную отправку email не делать: код подтверждения нужно эмулировать в модальном окне.
- Для регистрации нужна математическая CAPTCHA минимум с 3 арифметическими операциями.
- После отвязки модема в UI/CSV показывай `-1`, даже если в БД хранится `NULL`.
- Если реальный API для web-монитора недоступен, сделай mock/stub adapter с фиксированным контрактом.
- Для списков реализуй фильтр, пагинацию, экспорт CSV и корректное отображение количества записей.
- Деструктивные операции подтверждай модальным окном.

## Минимальный результат
- БД в 3НФ на разумном уровне;
- desktop: вход, регистрация, Главная, Торговые автоматы, Компании;
- web: Монитор ТА;
- 10 ручных тест-кейсов;
- 15+ unit-тестов для авторизации и регистрации;
- README с инструкцией запуска.

## Что нельзя пропускать
- валидацию email и пароля;
- CAPTCHA;
- проверку кода франчайзи;
- экспорт CSV;
- сортировки и фильтры web-монитора;
- поведение отвязки модема;
- тесты.
